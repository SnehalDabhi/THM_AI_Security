# TryHackMe AI Security Course — Portfolio

This repository documents my journey through the **AI Security labs on TryHackMe (THM)**.  
It’s more than a set of notes — it’s a portfolio of case studies showing how I approach **AI threat modeling, vulnerability analysis, and defensive design**.

---

## Why This Repository Matters
AI systems are powerful, but they’re also vulnerable. Through these labs, I explored how attackers exploit large language models and how defenders can respond.  

This repository highlights:
-  **Proof of execution** — screenshots and writeups that demonstrate real, hands‑on work.  
-  **Framework mapping** — every lab is tied to **OWASP LLM Top 10, STRIDE, and MITRE ATLAS**.  
-  **Security mindset** — I don’t just identify risks; I propose mitigations that strengthen systems.  
-  **Portfolio consistency** — this repo is part of a broader collection of projects I maintain, each dedicated to a specific challenge or room.  

---

## Course Coverage
The TryHackMe AI Security course guided me through labs on:
- **Prompt Injection** → How malicious inputs bypass guardrails.  
- **RAG Security** → Risks in retrieval‑augmented generation pipelines.  
- **Data & Model Poisoning** → Manipulating training pipelines and knowledge bases.  
- **Sensitive Information Disclosure** → Preventing leakage of secrets and PII.  
- **Improper Output Handling** → Securing frontends against unsafe LLM outputs.  
- **Supply Chain Risks** → Identifying vulnerabilities in datasets, libraries, and registries.  

Each lab is documented in its own folder with a **writeup.md** (my case study) and supporting **screenshots**.
